# No Top-of-File Comments

Do not add banner/preamble comments at the top of a source file — the "this file does X" summary, a port note, or leftover narration placed above the first import or declaration. They add noise, drift out of sync with the code, and accelerate context compaction.

This is enforced by the `block-top-of-file-comments.py` PreToolUse hook, which blocks an edit that **introduces** a leading comment into a file that didn't already have one. Files that already open with a comment are grandfathered — the hook only guards currently-clean files — but don't add new ones.

## What counts as a top-of-file comment

The first non-blank line of the file being a comment, once the exempt leading lines below are skipped:

- Python: `#` line comments and `"""..."""` string banners that are **not** the module docstring
- JS/TS/Dart: `//` line comments and `/* ... */` block comments

## Exempt leading lines (never blocked)

- **Shebangs** — `#!/usr/bin/env python3`
- **License / copyright headers** — a leading comment whose text contains `copyright`, `license`, `SPDX-License-Identifier`, or `@license`
- **Lint / type pragmas** — `# noqa`, `# type: ignore`, `# ruff:`, `# mypy:`, `# pylint:`, `// eslint-disable`, `/* eslint-disable */`, `// @ts-nocheck`, `// @ts-expect-error`, `// prettier-ignore`
- **Generated-file markers** — a leading comment containing `@generated`, `auto-generated`, `do not edit`, or `code generated by`
- **Python module docstrings** — a `"""..."""` (or `'''...'''`) string literal as the first statement is a real language construct, not a comment, and stays allowed

The hook skips these lines first, then checks whether what remains at the top is a disallowed comment.

## Rationale

Comment only where the *why* is non-obvious — a hidden constraint, a subtle invariant, a workaround. A comment that restates what the file is (`// Zustand auth store.`) is exactly the noise this rule removes. Put purpose in names and structure, not a preamble.
